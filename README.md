# YandexTestTask

## Описание

Проект предназначен для автоматизированного тестирования REST API сервиса Яндекс.Диск.

Тесты выполняют запросы к API:

```text
https://cloud-api.yandex.net
```

Проект проверяет работу основных HTTP-методов: `GET`, `POST`, `PUT`, `DELETE`. Во время выполнения тестов создаются, изменяются, копируются, скачиваются и удаляются тестовые файлы на Яндекс.Диске.

## Используемый стек

- Java 21
- Maven
- JUnit 5
- Rest Assured
- Jackson
- Allure JUnit 5
- JUnit Pioneer
- SLF4J Simple

## Учетные данные

### credentials.json

`credentials.json` содержит OAuth-токен тестового аккаунта Яндекс.Диска. Перед запуском тестов необходимо создать этот файл в папке src/main/resources и указать в нем OAuth-токен:

```json
{
  "token": "token_example"
}
```

## Реализованные тесты

В проекте реализованы проверки:

- получение информации о диске;
- создание папки;
- загрузка текстового файла;
- переименование файла;
- копирование файла;
- проверка размера и содержимого скопированного файла;
- проверка изменения занятого места на диске после загрузки файла;
- загрузка изображения и сравнение с локальным файлом;
- изменение расширения файла;
- перемещение файла в корзину;
- восстановление файла из корзины;
- удаление файла из корзины навсегда;
- получение статуса асинхронной операции;
- получение списка файлов;
- получение списка последних загруженных файлов.

## Запуск тестов

Запуск всех тестов:

```
mvn clean test
```

Запуск конкретного тестового класса:

```
mvn test -Dtest=DiskFileApiTest
```

## Allure-отчет

После выполнения тестов allure формирует отчеты в папке allure-results

Открыть отчет можно командой:

```
allure serve allure-results
```
